실험은 SSH 원격 환경에서 수행되므로 .sh 파일은 기본적으로 nohup + background 실행을 제공해야 한다.
SSH 연결이 끊겨도 학습/추론/분석 프로세스가 계속 진행되어야 하며, PID 와 log path 를 남겨야 한다.

현재 PSD 계열 bash 진입점 규칙은 아래처럼 고정한다.

- `psd_analysis` 공식 bash 진입점은 `bash/run_psd.sh` 하나다. `bash/psd.sh` 는 Linux launcher 가 아니라 Python/ML 인수 template 이며, 직접 실행되지 않도록 막아 둬야 한다.
- `bash/psd.sh` 는 기본 인수 틀만 남긴 내부 config template 이어야 한다. 하이퍼파라미터 값은 편집하기 쉽도록 비워 두되, 실제 실험은 `bash/run_psd.sh` 로만 시작한다.
- `bash/run_psd.sh` 는 한 번 실행으로 여러 scenario 를 병렬 실행하고, scenario parent 를 `<OUT_ROOT>/<EXP_NAME>/<scenario_index>/` 로 만든다. 각 scenario parent 아래 `log/` 에 PID / nohup log 를 저장한다.
- `dataset_psd` 공식 bash 진입점은 `bash/dataset_psd.sh` 하나다. 별도 `run_dataset_psd.sh` 를 남기지 않고, 이 단일 파일이 config template + nohup launcher 를 함께 맡는다.

bash 스크립트는 관련 run.py 의 실제 CLI 와 일치해야 한다.
즉 더 이상 존재하지 않는 인수는 쓰면 안 되고, 새로 추가된 주요 인수는 노출해야 한다.

PSD analysis 스크립트는 특히 아래 사항을 반영해야 한다.

- `--dataset`, `--model`, `--readout_mode` 를 현재 CLI 그대로 노출하고, `--dataset` 은 단일 인수이며 `--model`, `--readout_mode` 는 같은 dataset 안에서 조합별 직렬 실행이라는 설명을 bash 주석에 반영한다.
- 공식 canonical dataset 이름은 `s-mnist`, `dvsgesture`, `shd`, `deap`, `forda` 다.
- 공식 model 토큰은 기존 `lif` / `rf` 계열 PSD variant 외에 `tc`, `ts`, `dh`, `d_rf`, `my_*` 도 포함한다. fixed-branch dendritic 모델은 `dh_snn_2`, `d_rf_4` 처럼 suffix `_<숫자>` 를 허용하고, `my_*` 는 `S_min`, `S_max` 로 branch range 를 정한다는 설명을 bash 주석에 반영한다.
- 기본값은 전체 epoch 분석 저장이며, `--plot_epoch` 지정 시 해당 epoch 에서만 epoch root 와 선택 epoch 분석 산출물을 만든다는 설명을 bash 주석에 반영한다.
- readout mode 선택 인수 노출
- readout mode 에 여러 값을 주면 직렬 실행된다는 설명 유지
- output neuron 뒤에 learned NN head 가 없다는 설명 유지
- `final_membrane` 가 마지막 membrane 값을 raw pre-softmax score 로 쓰고, 이 mode 에서만 output layer spike/reset 이 비활성화된다는 설명 유지
- `max_rate` 가 평균 firing-rate 를 raw score 로 쓴다는 설명 유지
- `earliest_spike` 의 공식 supervised path 는 output layer spike / membrane record 를 저장한 뒤 released First-spike 계열 timing loss 를 적용하는 것이며, train/test accuracy 와 probe-set accuracy 도 같은 first-time prediction 규칙을 쓴다는 설명을 bash 주석에 반영한다.
- RF 는 exact ZOH 만 지원하며 Euler 관련 인수는 노출하면 안 된다.
- RF clip edge 는 Nyquist 가 0.5 인 normalized frequency 단위라는 설명 유지
- LIF / RF 동역학 학습 파라미터 초기화가 bounded uniform physical-domain policy 라는 점을 bash 주석에 드러내고, clip model 에서는 같은 분포를 public CLI clip 구간으로 좁혀 적용한다는 점을 적는다. 현재 RF clip CLI 는 frequency 만 제한하므로 RF damping magnitude 는 free bounded-uniform 초기화를 유지한다고 설명한다.
- RF 저장 통계는 `rho` 와 `f_cyc_per_sample` 이라는 설명 유지
- probe set mean PSD waveform 은 window 없는 simple periodogram exact plot 이라는 설명 유지
- periodogram heatmap 은 exact periodogram 의 userbin 집계라는 설명 유지
- mean spectrogram 은 window 없는 exact sliding simple periodogram 이라는 설명 유지
- spectrogram heatmap 은 exact spectrogram 을 userbin 으로 집계한 frame-major 그림이라는 설명 유지
- 각 PSD bundle 은 raw / centered 선형 8개 PNG + 대응 dB 8개 PNG + `summary.json` 을 저장한다는 설명 유지
- 선택된 epoch root 에 `attenuation_stats/` 와 `all_layers_summary.csv` 가 저장되고, 최종 snapshot 은 `training_complete_stats/attenuation_stats/` 와 `training_complete_stats/all_layers_summary.csv` 에도 유지된다는 설명을 bash 주석에 반영한다
- 학습 중에는 selected epoch / probe reference PNG 를 직접 그리지 않고 plot payload 를 process-local CPU 메모리에만 홀드한 뒤, 학습 완료 후 tqdm 로그와 함께 렌더링하고 성공한 payload 를 즉시 메모리에서 제거한다는 설명을 bash 주석에 반영한다.
- accuracy 는 CSV 와 그 CSV 기반 단일 plot 으로 저장된다는 설명 유지
- output layer membrane / spike 저장 설명 유지
- 선택된 epoch root 에 `probe_set_accuracy.txt`, hidden layer `w_plot.png`, grouped 시나리오의 hidden-layer block PSD bundle / block weight plot 이 저장되고, `training_complete_stats/probe_set_accuracy/` 아래에 최종 probe-set accuracy 가 따로 저장된다는 설명을 bash 주석에 반영한다
- `epochs = 0` 이면 학습 및 epoch 분석을 수행하지 않는다는 설명을 bash 주석에 반영한다. input probe reference 영구 저장은 `dataset_psd` 의 `--probe_plot` 로만 설명한다.
- derivative semi-metric 은 probe input reference 와 hidden / output bundle 의 같은 family mean plot 8개 사이에서만 추적하고, `element_*` heatmap 은 제외하며, 결과를 `training_complete_stats/derivative_semi_metric/` 아래 저장한다는 설명을 bash 주석에 반영한다.
- plot writer worker / queue / DPI 환경변수 설명 유지
- probe set 은 split 이름 + seed 로 정한 label별 canonical order 의 prefix 로 결정되고, requested probe-sample counts 는 prefix 길이만 바꾼다는 설명 유지
- input probe reference 저장 on/off 는 `dataset_psd` 의 `--probe_plot` 하나로만 설명하고, `psd_analysis` bash 에는 관련 CLI 를 노출하지 않는다.
- 공식 exact 경로에서 의미가 없는 legacy 인수는 기본 bash 변수로 노출하지 않는다. 특히 `--window_fn` 은 기본 스크립트 변수로 두지 않는다.
- bash 변수 주석은 빈 배열이면 해당 CLI 인수를 생략한다는 점과, 공백 분리 배열 인수를 어떻게 주는지까지 바로 보이게 적는다.

dataset_psd 스크립트는 특히 아래 사항을 반영해야 한다.

- `--dataset`, `--data_root`, `--out_root`, `--gpu`, `--batch_size`, `--download` 를 현재 CLI 그대로 노출한다.
- 공식 canonical dataset 이름은 `s-mnist`, `dvsgesture`, `shd`, `deap`, `forda` 다. `--dataset` 은 단일 인수라는 설명을 bash 주석에 반영한다.
- 더 이상 `--h5_path`, `--spike_filter_type`, `--spike_filter_tau`, `--save_filtered_input` 같은 인수는 쓰지 않는다.
- `--shd_*`, `--dvsgesture_*`, `--deap_*` 같은 dataset별 인수는 현재 CLI 와 일치하게 노출한다.
- 선택된 dataset 의 train / test split 을 각각 처리한다는 설명을 유지한다.
- split 당 raw / centered 선형 8개 PNG, 대응 dB 8개 PNG, `summary.json` 을 저장한다는 설명을 유지한다.
