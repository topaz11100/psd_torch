Origin/ 아래 저자 제공 코드는 읽기 전용이다. `DH_SNN_neuron.py`, `TC_LIF_neuron.py`, `TS_LIF_neuron.py`, `D_RF_neuron.py` 처럼 paper/ 와 Origin/ 이 함께 있는 기존 뉴런 구현은 저자 제공 코드를 거의 그대로 유지하는 thin wrapper 여야 하며, 추가 로직은 프로젝트 공통 입출력 API, record hook, readout 호환에 필요한 최소 adapter 로 제한한다. 단 `RF_neuron.py` 는 상위 PSD 실험 명세가 vanilla RF 의 exact ZOH 이산화를 공식 규칙으로 고정하므로 그 범위 안에서 최소 재구현을 허용한다.

이 경로에는 paper/ 의 제안 구조 중 neuron layer 구현만 둔다.
나중에 이 neuron layer 들을 import 해서 제안 구조 중심의 모델을 조립할 수 있어야 한다.

따라서 아래 파일들이 존재해야 한다.

- LIF_neuron.py
- RF_neuron.py
- TC_LIF_neuron.py
- TS_LIF_neuron.py
- DH_SNN_neuron.py
- D_RF_neuron.py
- my_DH_SNN_neuron.py
- my_D_RF_neuron.py
- my_R_DH_SNN_neuron.py

이 경로의 파일은 hidden neuron layer 구현을 담당한다.
공통 output functional readout 과 PSD analysis 전용 output spike 기록 로직은 `src/common/readout.py`, `src/common/snn_builder.py`, `src/common/first_spike_loss.py` 에 둘 수 있다. output neuron 뒤에 learned classifier head 를 추가하지 말고, 실제 output neuron layer 의 membrane / spike sequence 에 대해 readout 규칙만 적용한다. 단 `final_membrane` 에서만 output layer 의 spike emission 과 spike-triggered reset path 를 비활성화하고, `earliest_spike` / `max_rate` 는 일반 spiking output layer 를 유지한다. `earliest_spike` 의 공식 supervised path 는 output spike / membrane record 기반 First-spike timing loss 다. `RF_neuron.py` 는 real-split + exact ZOH 구현을 유지하고, RF clip 은 normalized frequency `[0, 0.5]` 입력을 omega 로 변환해 적용한다. `LIF_neuron.py`, `RF_neuron.py` 의 동역학 학습 파라미터 초기화는 fan-based Kaiming/Xavier 가 아니라 유효 물리 파라미터 구간에서의 bounded uniform sampling 이어야 하며, clip variant 는 같은 규칙을 유지하되 각 neuron clip interval 안으로 support 를 좁혀야 한다. 현재 RF clip 은 frequency 만 public CLI 로 제한하므로 damping magnitude 는 free bounded-uniform 초기화를 유지한다.

추가 호환 규칙은 아래와 같다.

- neuron layer 구현은 `paper/proposed/psd_analysis.md` 의 output-layer 규칙과 모순되면 안 된다.
- PSD raw / centered dual 저장, windowless exact periodogram / spectrogram, `probe_set_accuracy.txt` 저장은 공통 PSD 분석 계층에서 처리하며, neuron 구현은 이를 방해하는 별도 classifier head 나 우회 readout 을 추가하면 안 된다.
