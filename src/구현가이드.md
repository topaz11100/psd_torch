# src/ 재구현 가이드 (초보자용)

이 문서는 `promft.txt`, `structure.txt`, `paper/proposed/*.md` 를 기준으로 현재 `src/` 구조를 빠르게 이해하도록 돕는 안내서입니다. 명세 충돌 시에는 반드시 `paper/proposed/psd_analysis.md` 를 우선합니다.

## 1) 핵심 원칙

1. **논문 + 저자 공개코드(Origin)가 있는 뉴런은 자의적으로 다시 쓰지 않는다.**
2. **PSD / spectrogram / probe-set / 저장 규칙은 proposed 명세에 맞춘다.**
3. **초보자가 따라갈 수 있게 파일 역할을 분리한다.**
   - dataset별 전처리는 dataset 파일로 분리
   - dataset 선택/별칭/metadata 조립은 registry로 분리
   - plot 저장은 plot 모듈로 분리
   - 학습/실험 orchestration은 util driver에서만 담당
4. **과거 monolithic 파일은 compatibility export만 남긴다.**

---

## 2) 폴더별 역할

- `src/data/`
  - dataset별 로더와 전처리 어댑터
- `src/neurons/`
  - Origin thin wrapper + proposed 전용 뉴런 구현
- `src/readout/`
  - `final_membrane`, `first_spike`, `max_rate` readout
- `src/model/`
  - 모델 조립, optimizer, 학습 step
  - `first_spike_loss.py`: Origin First-spike released code thin wrapper
- `src/signal/`
  - exact periodogram / spectrogram / PSD payload 계산
- `src/stat/`
  - probe selection, curve-shape semi-metric 같은 통계 규칙
- `src/plot/`
  - deferred plot payload 저장과 최종 렌더링
- `src/util/`
  - 위 폴더에 분류되지 않는 기타 공통 파일들
- `src/dataset_psd.py`
  - dataset input 기준선 PSD 실험 메인
- `src/psd_analysis.py`
  - 모델 학습 + hidden/output 분석 메인

## 6) PSD 분석 구현에서 중요한 구조 포인트

- input probe reference 영구 저장 책임은 `dataset_psd`
- `psd_analysis` 는 같은 deterministic probe set 을 공유하지만 내부 비교 payload 로 사용
- hidden / output layer bundle 은 selected epoch 에서만 epoch 디렉터리 생성
- 학습 중에는 PNG 를 바로 그리지 않고 numeric payload 만 들고 있다가 학습 완료 후 일괄 렌더링
- `training_complete_stats/shape_sim_metric/` 에 최종 **centered pointwise L2 semi-metric** 요약을 저장

---

## 7) 초보자 추천 코드 읽기 순서

1. `src/psd_analysis.py` (CLI 인자)
2. `src/util/psd_analysis_driver.py` (실험 orchestration)
3. `src/model/snn_builder.py` (`build_layer`, `build_snn_classifier`)
4. `src/neurons/*_neuron.py` (실제 1-step 뉴런 동역학)
5. `src/readout/readout.py` (최종 판정 규칙)
6. `src/signal/fft_analysis.py`, `src/signal/psd_utils.py` (PSD 계산)
7. `src/data/registry.py` + 필요한 dataset 파일 하나

이 순서로 보면 `CLI -> dataset/model 조립 -> forward -> readout -> PSD 저장` 흐름이 자연스럽게 보인다.


## 8) 주석 정책

- `src/` 아래 모든 함수에는 초보자 기준의 설명 docstring 을 붙인다.
- Origin thin wrapper는 내부 구현을 길게 다시 설명하지 않고, 어느 논문의 어느 파일을 그대로 가져왔는지만 명시한다.
